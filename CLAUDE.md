# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
bun dev          # start dev server
bun build        # production build
bun lint         # ESLint
bun format       # Prettier (write)
```

No test suite is configured.

## Stack

- **TanStack Start** (SSR framework) + **TanStack Router** (file-based routing)
- **React 19** + **TypeScript**
- **Tailwind CSS v4** (CSS-first config — no `tailwind.config.ts`)
- **Motion** (`motion/react`) for all animations
- **shadcn/ui** components in `src/components/ui/` (Radix-based, auto-generated — don't hand-edit)
- **Bun** as package manager and runtime

## Architecture

### Routing
Routes live in `src/routes/`. File names map directly to URLs (see `src/routes/README.md`). `__root.tsx` is the app shell wrapping every page. `routeTree.gen.ts` is auto-generated by the router plugin — never edit it manually.

Currently the site is a single-page portfolio at `/` (`src/routes/index.tsx`). All sections (Nav, Hero, Marquee, Work, About, Contact, Footer) are co-located in that file as named functions.

### Design system & tokens
All brand tokens are CSS custom properties defined in `src/styles.css`. The palette is:
- `--plum` (#4A365F) → `primary`
- `--teal` (#264653) → used directly as `var(--teal)`
- `--gold` (#D6B36A) → `accent`
- `--ivory` (#FAF8F5) → `background`
- `--surface` (#EDE8F8) → `secondary`

Tailwind maps these via `@theme inline` in `styles.css`. Use the semantic tokens (`text-primary`, `bg-secondary`, etc.) rather than raw palette names in components.

Typography: `font-display` = Fraunces (serif, loaded from Google Fonts per-route in `head()`), `font-sans` = Inter.

### Animation
`src/components/SectionReveal.tsx` exports a `<Reveal>` component — a scroll-triggered fade+slide using `whileInView`. Use it to wrap any section or block that should animate in on scroll. It respects `prefers-reduced-motion` via `useReducedMotion()`.

### Server functions
`src/lib/api/example.functions.ts` shows the pattern for TanStack Start server functions. `src/lib/config.server.ts` is server-only config (never import from client components).

### Assets
Static assets referenced in code use `.asset.json` files (e.g. `src/assets/av-logo.jpg.asset.json`) — a Lovable convention that resolves to a hosted URL at `asset.url`.
